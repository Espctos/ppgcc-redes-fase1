\begin{abstract}
This paper presents an experimental comparative analysis between TCP and Reliable UDP
(R-UDP) implementing the Go-Back-N (GBN) sliding window protocol for file transfer
under degraded network conditions. The infrastructure relies on Python sockets running
inside Docker containers (Ubuntu~22.04), with network impairment injected via
\texttt{tc qdisc netem} across three scenarios: 0\%/10\,ms, 10\%/50\,ms, and
20\%/100\,ms of packet loss and latency. A cross-validation methodology confronts
application-level metrics (throughput, latency, retransmissions) against network-level
packet captures via \texttt{tcpdump}. Results show TCP reaching up to 487\,Mbps in
ideal conditions, while R-UDP/GBN achieves only 4.7\,Mbps due to cascaded timeout
retransmissions -- even with 0\% loss. Cross-validation reveals network overhead
between $+52\%$ and $+420\%$ above application bytes, dominated by retransmissions.
\end{abstract}

\begin{resumo}
Este artigo apresenta uma análise experimental comparativa entre TCP e UDP Confiável
(R-UDP) com janela deslizante Go-Back-N (GBN) para transferência de arquivos em
condições de rede degradadas. A infraestrutura utiliza sockets Python em contêineres
Docker (Ubuntu~22.04), com degradação injetada via \texttt{tc qdisc netem} em três
cenários: 0\%/10\,ms, 10\%/50\,ms e 20\%/100\,ms de perda e atraso. Uma metodologia
de validação cruzada confronta métricas de nível de aplicação (vazão, latência,
retransmissões) com capturas via \texttt{tcpdump}. Os resultados mostram o TCP
atingindo até 487\,Mbps em condições ideais, enquanto o R-UDP/GBN alcança apenas
4,7\,Mbps devido a retransmissões em cascata por timeout -- mesmo com 0\% de perda.
A validação cruzada revela overhead de $+52\%$ a $+420\%$, dominado pelas retransmissões.
\end{resumo}

\section{Introdução}

O desempenho de protocolos de transporte em redes com perdas é tema central em Redes
de Computadores. O TCP oferece entrega confiável via AIMD, mas reage fortemente a
perdas~\cite{kurose2017}. O Go-Back-N (GBN) retransmite toda a janela ao detectar
um erro~\cite{tanenbaum2011}, sendo mais simples que o Selective Repeat ao custo de
maior desperdício de banda sob perdas.

Objetivos: (i)~implementar TCP e R-UDP/GBN em Python com Docker; (ii)~injetar
degradação via \texttt{tc qdisc netem}; (iii)~confrontar métricas de aplicação e
rede; (iv)~quantificar overhead e impacto das retransmissões em cascata.

\section{Trabalhos Relacionados}

O QUIC~\cite{iyengar2021} supera o GBN retransmitindo apenas o pacote perdido.
O TCP CUBIC~\cite{ha2008} suaviza a degradação em redes de alta largura de banda.
A emulação via \texttt{netem}~\cite{hemminger2005} combinada com \texttt{tcpdump}
para validação cruzada~\cite{paxson1997} é prática recomendada.

\subsection{Protocolo R-UDP com Go-Back-N}

Formato do pacote:
\begin{center}
\texttt{| seq (4B) | flags (1B) | win\_base (4B) | CRC32 (4B) | payload ($\leq$1400\,B) |}
\end{center}

Cabeçalho: 13~bytes. Checksum: CRC32. Flags: \texttt{DATA}~(0x01),
\texttt{ACK}~(0x02), \texttt{FIN}~(0x04), \texttt{NAK}~(0x08).
Janela $W=8$, timeout $T=0{,}3$\,s. Timer único no pacote base; ao expirar,
retransmite janela inteira.

\begin{lstlisting}[language=Python, caption={Núcleo Go-Back-N Sender}]
while True:
    while next_seq <= total and next_seq < base + W:
        send_packet(next_seq); next_seq += 1
    if base > total: break
    if time.perf_counter() - timer_start > TIMEOUT:
        for s in range(base, next_seq):
            send_packet(s); retransmissions += 1
        timer_start = time.perf_counter()
\end{lstlisting}

\subsection{Coleta e Validação Cruzada}

Duas fontes simultâneas por experimento: (1)~\textbf{Aplicação} -- tempo, vazão
e retransmissões; (2)~\textbf{Rede} -- \texttt{.pcap} convertido a CSV via Scapy.
5~repetições por protocolo/cenário (30~execuções).
$\text{overhead}(\%) = \frac{\text{bytes}_\text{tcpdump} - \text{bytes}_\text{app}}{\text{bytes}_\text{app}} \times 100$.

\section{Conclusão}

Este trabalho apresentou análise experimental comparativa entre TCP e R-UDP/GBN
com validação cruzada em três cenários de degradação. Resultados: (1)~TCP supera
GBN por $\sim103\times$ no Cenário~A por 28.032 retransmissões espúrias;
(2)~GBN apresenta latência $3{,}4\times$ superior no Cenário~C; (3)~retransmissões
superam previsão analítica em 18\%; (4)~overhead varia de $+52\%$ a $+420\%$.

Como trabalho futuro: timeout dinâmico via EWMA e simulação SimPy (Fase~2).
